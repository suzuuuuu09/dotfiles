# 質問するときのガイド

ユーザーに質問する前に、このガイドを読む。

## 質問する前に確認すること

- リポジトリ内の関連ファイル、設定、履歴で確認できる事実は、先に自分で調べる。
- 安全で小さい既存パターンがある場合は、それを採用し、質問を増やさない。
- 質問が必要なのは、ユーザーの意図・優先順位・権限など、調査しても決められない判断だけである。

## 質問のしかた

- 複数の質問は Q1, Q2 のように番号をつけて、順番に答えやすくする。
- 選択肢はA, B, Cのように示し、必ず推奨する選択肢も明記する。各選択肢の影響と推奨理由を簡潔に示す。
- 回答しやすい具体的な質問にする。事実確認をユーザーに丸投げしない。
- 回答がないと進めない場合だけ、作業を止めて質問する。そうでなければ安全な前提を明示して進める。
- 確認は Yes/No/OKで答えられるclosed questionにする。
    - ✅「このライブラリの安定板を使いますか？」
    - ❌「このライブラリはどのバージョンを使いますか？」

## 回答後

- 回答内容を要約して、変更の範囲と前提に反映する。
- 追加の判断が必要なら、次の質問は前の判断を踏まえてから一つだけ尋ねる。
