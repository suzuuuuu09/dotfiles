# エージェントガイドライン
## 環境

- GitHub: suzuuuuu09
- リポジトリ: ghqで管理 (`~/ghq/github.com/owner/repo`)

## コーディング

- 関心の分離を保つ
- 状態とロジックを分離する
- 可読性と保守性を重視する
- 既存のコードスタイルを優先する
- 大きな変更の前に方針を説明する
- 過度に抽象化しすぎない
- 変更範囲を必要最小限にする
- 既存の命名規則・ディレクトリ構成に合わせる

## 作業方針

- 作業前に`find-docs`スキルで最新ドキュメントを取得し、それを参考に作業する
- 作業前に関連ファイルを確認する
- 推測で実装せず、既存コードの実装パターンを優先する
- 仕様が曖昧な場合は、最も安全で小さい変更を選ぶ
- 破壊的な操作を行う前に確認する
- 不要な依存関係を追加しない
- 依存関係を追加する場合は理由を説明する

## Git

- ユーザーに依頼されない限り、commit / push / tagは使用しない
- 既存の変更がある場合、自分の変更と混ぜない
- コミットする場合は、変更内容を確認して行う

## 禁止事項

- 不要な大規模リファクタリングはしない
- `.env`をコミット・閲覧・編集しない
- `rm -rf`などの破壊的操作を勝手に実行しない
- 関係ないファイルは変更しない
